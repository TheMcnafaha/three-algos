Intro

This is a quick example of how different search algorithms perfoms in terms of steps. The test will involve you answering the logic for the algorithms evertime they need to make a step while the computer program keeps track of how many steps each algorithm has done so far.

A step is defined as the any time the algorithm begins to search again (for a more nerdy defenition, a step is any time the algorithm enters or reenters a loop or function). Thus, the fastest algorithm will be the one that takes the least steps (or searches) to find your number from a list of numbers.

For example: You are given a list of all the integers from 0 to 100 (i.e a number list from 0 to 100)and want to find the number 59. If algorithm A takes 59 steps to find your number while algorithm B takes 7 steps, algorithm B would be declared the faster algoy
